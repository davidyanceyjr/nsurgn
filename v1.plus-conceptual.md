# nsurgn v1.plus Conceptual Command Set

Status: conceptual aside
Relationship to v1.0: explicitly out of scope for the v1.0 foundation release

This document preserves ideas for commands that may build on the v1.0 artifact model after discovery, grouping, leader selection, classification, target resolution, and reporting are proven.

Nothing in this document is a v1.0 commitment.

---

## 1. Discovery Expansion

### `nsurgn all`

List every visible artifact, including host-classified groups and low-scoring groups.

Synopsis:

```text
nsurgn all [--group <mode>] [--format <format>]
```

### `nsurgn scout`

Run a short suspicion-oriented scan that highlights unusual namespace or cgroup layouts.

Synopsis:

```text
nsurgn scout [--format <format>]
```

### `nsurgn tree`

Show artifact and PID namespace relationships as a hierarchy.

Synopsis:

```text
nsurgn tree [<artifact-id|pid>] [--format <format>]
```

## 2. Deeper Inspection

### `nsurgn mounts`

Show mount namespace and mountinfo details for an artifact leader.

Synopsis:

```text
nsurgn mounts <artifact-id|pid> [--format <format>]
```

### `nsurgn exe`

Inspect or extract the executable referenced by `/proc/<pid>/exe`.

Synopsis:

```text
nsurgn exe <artifact-id|pid>
nsurgn exe <artifact-id|pid> --extract <host-dest>
```

### `nsurgn fds`

Inspect selected file descriptors for an artifact or process.

Synopsis:

```text
nsurgn fds <artifact-id|pid> [--leader-only] [--format <format>]
```

### `nsurgn env`

Inspect process environment data with explicit secret-exposure warnings.

Synopsis:

```text
nsurgn env <artifact-id|pid> [--leader-only] [--redact]
```

## 3. Path Inspection

These commands would resolve paths relative to an artifact target root.

### `nsurgn ls`

List a path inside an artifact root filesystem.

Synopsis:

```text
nsurgn ls <artifact-id|pid> <target-path>
```

### `nsurgn cat`

Print a file from an artifact root filesystem.

Synopsis:

```text
nsurgn cat <artifact-id|pid> <target-path> [--max-bytes <n>]
```

### `nsurgn stat`

Print metadata for a path inside an artifact root filesystem.

Synopsis:

```text
nsurgn stat <artifact-id|pid> <target-path>
```

### `nsurgn exists`

Check whether a path exists inside an artifact root filesystem.

Synopsis:

```text
nsurgn exists <artifact-id|pid> <target-path>
```

### `nsurgn checksum`

Calculate a checksum for a file inside an artifact root filesystem.

Synopsis:

```text
nsurgn checksum <artifact-id|pid> <target-path> [--sha256|--sha512|--md5]
```

## 4. Filesystem Operations

These commands are intentionally separated from v1.0 because they rely on correct target resolution and carry host-damage risk if implemented poorly.

### `nsurgn install`

Copy a host file or directory into an artifact root filesystem.

Synopsis:

```text
nsurgn install <artifact-id|pid> <host-src> <target-path> [--parents] [--overwrite]
```

### `nsurgn inject`

Alias for `install`. The name must be documented carefully: it means filesystem copy only.

Synopsis:

```text
nsurgn inject <artifact-id|pid> <host-src> <target-path> [--parents] [--overwrite]
```

### `nsurgn extract`

Copy a file or directory from an artifact root filesystem to the host.

Synopsis:

```text
nsurgn extract <artifact-id|pid> <target-path> <host-dest> [--overwrite] [--preserve]
```

### `nsurgn remove`

Remove a file or directory from an artifact root filesystem with explicit force requirements.

Synopsis:

```text
nsurgn remove <artifact-id|pid> <target-path> --force [--recursive]
```

### `nsurgn move-in`

Move a host file into an artifact root filesystem.

Synopsis:

```text
nsurgn move-in <artifact-id|pid> <host-src> <target-path> --force
```

### `nsurgn move-out`

Move a file from an artifact root filesystem to the host.

Synopsis:

```text
nsurgn move-out <artifact-id|pid> <target-path> <host-dest> --force
```

## 5. Process Operations

### `nsurgn signal`

Send a signal to an artifact leader, or to all artifact processes with explicit bulk targeting.

Synopsis:

```text
nsurgn signal <artifact-id|pid> <signal>
nsurgn signal <artifact-id|pid> <signal> --all
```

---

## 6. Namespace Entry

### `nsurgn enter`

Run an explicit command in selected namespaces of the artifact leader using `nsenter`.

Synopsis:

```text
nsurgn enter <artifact-id|pid> [options] -- command...
```

Candidate options:

```text
--mount
--uts
--ipc
--net
--pid
--user
--cgroup
--time
--root
--wd <path>
```

Design constraints:

- Require `--` before the command.
- Do not enter the user namespace by default.
- Do not build shell command strings.
- Do not provide a custom chroot fallback.
- Treat `nsenter` as an explicit dependency for this command only.

---

## 7. Case Packaging and Comparison

### `nsurgn bundle`

Collect read-only artifact evidence into a host-side case directory.

Synopsis:

```text
nsurgn bundle <artifact-id|pid> <host-dir>
```

### `nsurgn diff`

Compare two artifact reports or live artifacts.

Synopsis:

```text
nsurgn diff <artifact-a> <artifact-b>
```

---

## 8. Promotion Rule

A command should move from this conceptual document into a future release spec only after the foundation can prove:

- target resolution is correct,
- leader selection is explainable,
- classification risk is visible,
- permissions and procfs restrictions are handled,
- output and errors are stable,
- safety behavior is testable.
